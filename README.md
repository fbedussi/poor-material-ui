# Poor man's Material UI

Recently I removed Material UI from a simple project where such an heavy dependency didn't make sense. I replaced it with some simple CSS rules and React components that keep the basic user experience intact with way less code.

Since it can be handy for other projects as well I extracted the components and the style in this separate project. Of course it contains only a small subset of all the Material UI goodies. But more can be added when needed, following the same principle of leveraging the native web platform APIs as much as possible.
